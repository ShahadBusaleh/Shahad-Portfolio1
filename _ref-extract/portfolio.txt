# AI Meeting Assistant (MOM) — Project Portfolio

**Organization:** Al Gosaibi
**Product:** Minutes of Meeting (MOM) — AI Meeting Assistant
**Production URL:** `https://itonline.algosaibi.sa/mom/`
**Role:** Enterprise integration, deployment, and full-stack delivery

---

## Executive Summary

The **AI Meeting Assistant (MOM)** is an enterprise web application that lets employees record meetings in the browser, capture live transcription, generate AI summaries, produce formal **Minutes of Meeting (MOM)** documents, and distribute them via PDF and Microsoft Graph email — all inside the company **IT Online Portal** with **single sign-on** through Microsoft Entra ID.

The system was designed for a **two-server Windows architecture**: an ASP.NET portal server handles authentication and YARP reverse proxy; a dedicated MOM server runs Next.js, FastAPI, LM Studio, and SQLite. Employees never see a second login or a separate MOM hostname in the address bar.

---

## Business Problem

Manual meeting minutes are slow, inconsistent, and hard to share across departments. The organization needed a solution that:

- Works inside the existing employee portal (not a standalone public app)
- Respects corporate identity (Entra ID SSO)
- Produces formal MOM documents with attendees and action items
- Integrates with Outlook calendar and email
- Keeps transcription load off the central server where possible
- Runs on on-premise Windows infrastructure

---

## Solution Overview

| Capability | How it works |
|------------|----------------|
| **Record** | Browser `MediaRecorder` + microphone |
| **Transcribe** | Browser **SpeechRecognition** (Chrome/Edge) for live captions; Whisper WASM available in codebase for optional paths |
| **Summarize** | LM Studio on MOM server (`qwen2.5-3b-instruct` or configured model) |
| **Store** | SQLite on MOM server |
| **MOM document** | **Minutes of Meeting** template → structured editor → PDF |
| **Share** | Export page → Microsoft Graph mail |
| **Calendar** | Outlook integration via MS Graph (`/calendar` page) |
| **SSO** | Portal Entra ID → YARP injects `X-User-*` headers |

## Key Challenges & Resolutions

### 1. 502 Bad Gateway through the portal
**Cause:** MOM services bound to `127.0.0.1`; YARP on the portal could not reach them over the network.
**Fix:** Bind to `0.0.0.0`, firewall rules portal → MOM on 3118/5167, `Ensure-MeetingAssistantBindHost` in deployment scripts.

### 2. Portal crash (HTTP 500.30)
**Cause:** Corrupted `appsettings.json` on DMSSRV (truncated URLs, invalid JSON).
**Fix:** Restored valid YARP config with `mom-frontend` and `mom-api` routes; documented pool name `issue-return-system`.

### 3. Transcripts not saving after recording
**Cause:** Frontend/backend contract mismatch; `useRecordingStop` skipped DB save when backend already created meeting ID; silent failures.
**Fix:** Always persist segments via `POST /recording/save-transcripts`; lenient API validation; surface errors to user.

### 4. Missing templates and broken share/export
**Cause:** `/templates` empty or 404; wrong API paths on export page.
**Fix:** Implemented `/templates` with **Minutes of Meeting** + **General Summary**; corrected export and MOM editor routes for `/mom/` base path.

### 5. Dual-server confusion
**Cause:** Portal fixes applied on MOM server (and vice versa).
**Fix:** Runbooks separating DMSSRV (portal/YARP) from MOM server (services only).

### 6. SSO without second login
**Cause:** MOM must trust portal-injected headers.
**Fix:** YARP claim transforms + FastAPI middleware reading `X-Forwarded-By-Portal` and `X-User-*`.

### 7. Local AI on CPU-only server
**Cause:** Cloud LLM not primary path for all workloads.
**Fix:** LM Studio on MOM server with actionable `/api/health` diagnostics.

### 8. Microsoft Graph for email and calendar
**Cause:** Application permissions, redirect URIs, and token flows.
**Fix:** Azure app registration, Graph endpoints, calendar prep flow, backend mail as application.

## Core User Journey

1. Employee logs into **IT Online Portal** (Entra ID).
2. Opens **Meeting Assistant** → `https://itonline.algosaibi.sa/mom/`.
3. **Records** meeting (mic permission).
4. **Transcript** appears live (browser speech engine).
5. Stops recording → transcripts saved to SQLite.
6. **Generate Summary** — selects **Minutes of Meeting** or **General Summary** template.
7. **MOM editor** — attendees, minutes, action items at `/mom/mom/{meetingId}`.
8. **Share** → export page → email PDF via Graph.
